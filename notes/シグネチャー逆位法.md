# シグネチャー逆位法（signature reversal / connectivity map）

## 一言でいうと

**「病気で狂った遺伝子発現パターンを、逆向きに戻す薬ほど効く」** という仮説。
薬と疾患の発現シグネチャーが **逆相関（反対向き）** なら、その薬は治療候補、という考え方。

---

## 直感

疾患では、ある遺伝子群が「上がる／下がる」パターン（＝疾患シグネチャー）ができる。

- 疾患で **UP** した遺伝子を、薬が **DOWN** に戻す
- 疾患で **DOWN** した遺伝子を、薬が **UP** に戻す

これが理想的な「効く薬」。つまり **薬の効果ベクトルが疾患ベクトルの真逆を向いている** ほど良い。

```
疾患シグネチャー :  ↑ ↑ ↓ ↑ ↓   (geneA geneB geneC geneD geneE)
効く薬の効果    :  ↓ ↓ ↑ ↓ ↑   ← 全部逆 = 逆位（reversal）
効かない薬      :  ↑ ↑ ↓ ↑ ↓   ← 同じ向き = むしろ悪化
```

---

## 数式レベルの流れ

1. **疾患シグネチャー**をつくる
   例: CREEDS の「疾患 vs 正常」で各遺伝子の変化量ベクトル `d`
2. **薬シグネチャー**をつくる
   例: LINCS L1000 の「薬あり vs コントロール」で各遺伝子の変化量ベクトル `x`
3. **共通遺伝子**に揃えて、2つのベクトルの向きの一致度を測る
   - コサイン類似度 `cos(d, x)` や connectivity score を使う
4. **判定**
   - `cos(d, x) < 0`（負） → 逆位している → **治療候補**
   - `cos(d, x) > 0`（正） → 同方向 → 治療に不向き

> ポイント：見ているのは発現量そのものではなく **「向き（up/down のパターン）」**。
> だから「逆位（inversion / reversal）」と呼ぶ。

---

## ゼロから理解する：内積・長さ・コサイン

数式が分からなくても、**「符号を掛けて足す → 長さで割る」** の2ステップだけ。

### (a) シグネチャー = ベクトル = 数字の並び

遺伝子ごとの変化量を横に並べただけ。プラス＝疾患で上がった、マイナス＝下がった。

```
疾患 d = ( +2, +1, -3, +2, -1 )    ← geneA..geneE の変化量
薬   x = ( -2, -1, +3, -2, +1 )    ← 符号が全部逆 = 逆位
```

### (b) 内積 = 同じ遺伝子どうし掛けて全部足す

```
d · x = (+2)(-2)+(+1)(-1)+(-3)(+3)+(+2)(-2)+(-1)(+1) = -19  ← 大きな負
```

- 逆向き（+×- / -×+）→ 掛け算が**マイナス**
- 同じ向き（+×+ / -×-）→ 掛け算が**プラス**
- **内積が負 = 逆位 = 治療候補**。これがロジックの核。

### (c) ベクトルの長さ = 矢印の長さ = 原点からの距離

2遺伝子で `v=(3,4)` なら、横3・縦4の直角三角形の**斜辺**。ピタゴラスの定理で:

```
|v| = √(3² + 4²) = √25 = 5
```

遺伝子が増えても同じ：**各成分を2乗して全部足してルート**。

```
v=(2,1,3,2,1) → |v| = √(4+1+9+4+1) = √19 ≈ 4.36
```

意味は「その薬が全遺伝子をどれだけ大きく動かしたか（変化の総量）」。

### (d) コサイン = 内積を長さで割って -1〜+1 に正規化

内積は、動かす量が大きい薬ほど自動的に膨らむ（向きが同じでも大きくなる）。
そこで**両方の長さで割って大きさの影響を消す**と、純粋に向きだけが残る。

```
cos(d, x) = (d · x) / (|d| × |x|)
             └向きの情報┘   └大きさを打ち消す┘
```

| cos | 角度 | 意味 |
|-----|------|------|
| +1 | 0°（同じ向き） | 薬が疾患を悪化方向 |
| 0 | 90°（無関係） | 無相関 |
| -1 | 180°（真逆） | **完全な逆位＝理想の薬** |

> 割り算のおかげで、ドカンと動かす薬もちょっとだけ動かす薬も、同じ土俵で「逆位度合い」を比較できる。

## このプロジェクトでの検証結果（Wang 2022 Fig3a 再現）

- **承認薬×適応疾患のペア**なら、逆位（負のコサイン）に寄るはず、という検証をした。
- 結果：**transcriptome レベルのコサインはほぼ 0 で、承認ペアもうまく負に寄らなかった**。
  - がん +0.016 / 免疫 +0.022 と、分離がほとんど出ない。
- これは **論文の主張そのままの再現**：
  「生の transcriptome では逆位シグナルが弱く、**regulome（TF活性など上流に射影した空間）で初めて分離が出る**」。

### 3rd paper への含意
- **単純に遺伝子を部分集合化（glycogeneだけに絞る）しても逆位は効かない**（glycogene版でも分離せず）。
- → 生発現をそのまま使うのではなく、**射影 or スコアリング（重み付け・上流空間への変換）が必要**。
- これが epitope potential 第一版の設計動機になっている。→ [[HANDOFF]] のオープン項目3

## 用語メモ

| 用語 | 意味 |
|------|------|
| signature | 遺伝子の up/down パターン（ベクトル） |
| reversal / inversion（逆位） | 薬が疾患パターンを反対向きにすること |
| connectivity score | 逆位の強さの指標（CMap由来）。負が強いほど良い |
| regulome | 遺伝子発現を上流の制御因子（TF活性等）空間に射影したもの |
